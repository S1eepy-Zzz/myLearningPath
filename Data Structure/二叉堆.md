# 二叉堆

二叉堆本质是一颗完全二叉树，特征是父节点总是不大于（不小于）任意一个子节点，可以用来维护最小（大）值。

### 基础版本

##### 存储形式

作为一颗完全二叉树，二叉堆可以使用1~index的数组来储存，对于节点p，左子节点储存在2*p的位置，右子节点储存在2*p+1的位置。

> eg.
> 1$\rightarrow$2,3
> 2,3$\rightarrow$4,5,6,7
> ...

### 核心操作

> ...父节点总是不大于（不小于）任意一个子节点...这里以大根堆为例

如果我们要维护一棵二叉树，考虑其中存在一个破坏性质的节点，我们要做的就是将其与其父节点或子节点互换，重复此流程，最多也不过是移动至最低或最顶。

##### prepare：交换

    #define swap(a, b) do { a ^= b; b ^= a; a ^= b; } while(0)

##### func 1：上浮

    void swim(int n)
    {
        for (int i = n; i > 1 && heap[i] > heap[i / 2]; i /= 2)
            swap(heap[i], heap[i / 2]);
    }

##### func 2：下沉

    int son(int n) // 找到需要交换的那个子节点
    {
        return n * 2 + (n * 2 + 1 <= size && heap[n * 2 + 1] > heap[n * 2]);
    }
    void sink(int n)
    {
        for (int i = n, t = son(i); t <= size && heap[t] > heap[i]; i = t, t = son(i))
            swap(heap[i], heap[t]);
    }

### 基础功能

##### 插入元素

元素直接插入尾部，再上浮即可

    void insert(int x)
    {
    heap[++size] = x;
    swim(size);
    }

##### 删除堆顶值

直接删除根节点，会变成两个堆，难以处理，这里通过交换根节点与叶子节点，再删去叶子节点，从而保证对的结构大体不被破坏，再进行维护
void pop()
{
swap(heap[1], heap[size--]);
sink(1);
}

##### 查询

直接返回根节点即可。

    int top()
    {
        return heap[1];
    }

##### 建立

可以从一个原始无序数组建立堆，从底部到顶部依次下沉即可。实际上因为叶子节点不需要下沉，所以可以从
处开始遍历。

    void build(int A[], int n) // 从一个（这里是0-index的）数组O(n)地建立二叉堆
    {
        memcpy(heap + 1, A, sizeof(int) * n);
        size = n;
        for (int i = n / 2; i > 0; --i)
            sink(i);
    }

### 特别思路——对顶堆

这个问题可以被进一步抽象成：动态维护一个序列上第k大的数，k值可能会发生变化．．
对顶堆由一个大根堆与一个小根堆组成，小根堆维护大值即前k大的值（包含第k个），大根堆维护小值即比第k大数小的其他数．

这两个堆构成的数据结构支持以下操作：
维护：当小根堆的大小小于k时，不断将大根堆堆顶元素取出并插入小根堆，直到小根堆的大小等于k；当小根堆的大小大于k时，不断将小根堆堆顶元素取出并插入大根堆，直到小根堆的大小等于k；
插入元素：若插入的元素大于等于小根堆堆顶元素，则将其插入小根堆，否则将其插入大根堆，然后维护对顶堆；
查询第k大元素：小根堆堆顶元素即为所求；
删除第k大元素：删除小根堆堆顶元素，然后维护对顶堆；k值+1/-1：根据新的k值直接维护对顶堆．
显然，查询第k大元素的时间复杂度是O(1)的．由于插入、删除或调整k值后，小根堆的大小与期望的k值最多相差1，故每次维护最多只需对大根堆与小根堆中的元素进行一次调整，因此，这些操作的时间复杂度都是$O(logN)$的．
