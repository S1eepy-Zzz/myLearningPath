# <font color="green">并查集</font>

> 通过将一个集合用其中的一个点表示，清晰地表达归属关系的一种数据结构（是一种树）

### 最简单的基础版本

##### func 1: 初始化

    int fa[MAXN];
    inline void init(int n)//将每个点的父节点设为自身
    {
        for (int i = 1; i <= n; ++i)
            fa[i] = i;
    }

##### func 2: 查询

    int find(int x)//递归寻找父节点
    {
        if(fa[x] == x)
            return x;
        else
            return find(fa[x]);
    }

##### func 3: 合并

    inline void merge(int i, int j)
    {
        fa[find(i)] = find(j);
        //这里注意find(i)，表示两个集合之间的合并
    }

### 优化1：路径压缩

##### 基础版本的劣势

随着树的高度增加，find()函数的开销越来越大

##### 解决方案

减小树的高度 $\rightarrow$在查询的同时优化树的构成

##### func 3\*：查询（路径压缩）

    int find(int x)
    {
        if(x == fa[x])
            return x;
        else{
            fa[x] = find(fa[x]);  //父节点设为根节点
            return fa[x];         //返回父节点
        }
    }

实际上，上面代码也等价于
int find(int x)
{
return x == fa[x] ? x : (fa[x] = find(fa[x]));
}
<font color = "brown">=的优先级没有三元运算符高，要加括号。</font>

### 优化2：按秩合并

##### 仍然存在的缺陷

即使在路径压缩的情况下，我们构建的树也可能结构比较复杂，这是因为路径压缩只有在查询操作时进行合并，尤其是面对树与树之间的合并，深度依旧可能较大

##### 弥补方案

当一个深度大的树和一个深度小的树合并时，我们应当将深度大的树的根节点作为新树的根节点，因为这会让新树的深度较小
因此，我们利用rank[i]来记录以i作为根节点的树的深度，通过比较大小，实现上面思路的合并

##### func 1\*：初始化

    inline void init(int n)
    {
        for (int i = 1; i <= n; ++i)
        {
            fa[i] = i;
            rank[i] = 1;//多了这一步
        }
    }

##### func 2\*

    inline void merge(int i, int j)
    {
        int x = find(i), y = find(j);    //先找到两个根节点
        if (rank[x] <= rank[y])
            fa[x] = y;
        else
            fa[y] = x;
        if (rank[x] == rank[y] && x != y)
            rank[y]++;
            //如果深度相同且根节点不同，则新的根节点的深度+1
    }

### 注意

如果将路径压缩和按秩合并同时使用，会造成以下结果：
秩不再是准确的子树高，而是子树高的上界，因为路径压缩可能改变子树高。
还可以将秩定义成子树节点数，因为节点多的树倾向更高。无论将秩定义成子树高上界，还是子树节点数，按秩合并都是尝试合出最矮的树，并不保证一定最矮。
