### 0-1背包问题

> 有$n$个物品和一个容量为$W$的背包，第$i$个物品有重量$w[i]$和价值$v[i]$两种属性.
> 要求选若干物品放入背包使背包中物品的总价值最大且背包中物品的总重量不超过背包的容量。
> 每个物体只有取或不去，对应$0$和$1$，这类问题便被称为$0-1$背包问题。

定义 $dp[i][j]$为容量为$j$的背包在只考虑前$i$个物品时的最大价值$$dp[i][j]=\max (dp[i-1][j],dp[i-1][j-w[i]]+v[i])$$
显然，$i$可以选择用循环变量来体现，数组可以改为$dp[j]$，方程改写为$$dp[j]=\max (dp[j],dp[j-w[i]]+v[i])$$
代码主体如下：

```
for (int i = 1; i <= n; i++)
  for (int l = W; l >= w[i]; l--) f[l] = max(f[l], f[l - w[i]] + v[i]);
```

考虑第$i$个物品时的高容量背包时，会受到低容量背包影响，将会发生重叠所以我们要从高容量向低容量遍历，避免数据被覆盖。

### 完全背包问题

> 完全背包模型与0-1背包类似，与0-1背包的区别仅在于一个物品可以选取无限次，而非仅能选取一次。

我们只要把0-1背包代码略作修改

```
for (int i = 1; i <= n; i++)
    for (int l = w[i]; l <= W; l++)
      if (f[l - w[i]] + v[i] > f[l]) f[l] = f[l - w[i]] + v[i];
```

也就是允许第$i$个物品被放入多次。

### 多重背包问题

> 多重背包每种物品有$k_i$个，而非一个。

我们可以选择把它当成普通的0-1背包去做，但我们也可以选择二进制分组优化
注意到，在这相同的$k$件物品的选择上，程序进行了相当多的重复工作。
<font color="gray" size=1>$\star$选择其中的1与2或是选择其中的2与3没有任何区别</font>
设$k=R+\Sigma_{i=0}^n2^i$，其中$2^{n-1}\leq n\leq 2^n$
按上面每一项“打包”物品，下证“打包”后可以组成任意数量的物品：
$\forall m=1,2,\cdots ,k:$
对于$0\leq m\leq k-R$，显然可以。
对于$k-R< m\leq k$，类似上面表示出m-R即可。

### 混合背包问题

> 混合背包就是将前面三种的背包问题混合起来，有的只能取一次，有的能取无限次，有的只能取$k$次

```
for (循环物品种类) {
  if (是 0 - 1 背包)
    套用 0 - 1 背包代码;
  else if (是完全背包)
    套用完全背包代码;
  else if (是多重背包)
    套用多重背包代码;
}
```

或者把0-1背包看成选一个的多重背包，完全背包看成选$\lfloor\frac{W}{w[i]}\rfloor$个的多重背包

### 二元费用问题

> 有$n$个任务需要完成，完成第$i$个任务需要花费$t_i$分钟，产生$c_i$元的开支，现在有$T$分钟时间,$W$元钱来处理这些任务，求最多能完成多少任务．

这其实就是一个0-1背包问题，仅仅是多了一维费用，多开一维变量存储即可。

```
for (int k = 1; k <= n; k++)
  for (int i = m; i >= mi; i--)    // 对经费进行一层枚举
    for (int j = t; j >= ti; j--)  // 对时间进行一层枚举
      dp[i][j] = max(dp[i][j], dp[i - mi][j - ti] + 1);
```

### 分组背包问题

> 给定N组物品和一个容量为V的背包。每组物品包含若干个，但在同一组内，你最多只能选择一件物品。

我们只需要对组进行0-1背包问题（将每个组视为一件价格，重量会变的物品），我们只要记录下每个变化的值，总体依然会保持最优解。

```
for (int k = 1; k <= ts; k++)          // 循环每一组
  for (int i = m; i >= 0; i--)         // 循环背包容量
    for (int j = 1; j <= cnt[k]; j++)  // 循环该组的每一个物品
      if (i >= w[t[k][j]])             // 背包容量充足
        dp[i] = max(dp[i],
                    dp[i - w[t[k][j]]] + c[t[k][j]]);  // 像0-1背包一样状态转移
```

在上面代码中，$t[k][i]$表示第$k$组的第$i$件物品的编号，$cnt[k]$表示第$k$组物品数量。

### 依赖背包问题

> 有些物品是从属于某个主件物品的附件，要买这个物品，必须购买它的主件。

按照\{主件，主件+附件\}的分组方式，当作分组背包问题。<font color="gray" size=2>若附件相同，可以进行二进制分组优化。</font>
